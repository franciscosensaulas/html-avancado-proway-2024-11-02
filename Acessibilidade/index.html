Atributos ARIA

O que é ARIA? ARIA (Acessible RICH Internet Applications) é uma especificação que define como tornar a web mais acessível para pessoas com deficiências. Ele fornece atributos adicionais para tornar a interação com elementos da página mais clara para tecnologias assistivas, como leitores de tela. O uso correto de ARIA melhora a navegação, facilita a compreensão de conteúdo dinâmico e contribui para uma experiência mais inclusiva na web.

  • arial-label: Define um rótulo acessível para um elemento, útil quando o texto vísivel não é suficiente
  • aria-hidden: Esconde um elemento da árvore acessível, informando aos leitores de tela para ignorá-lo.
  • aria-required: Marca um campo de form como obrigatório, permitindo que leitores de tela avisem os usuários
  • aria-expanded: Inidica o estado de expansão de um menu ou painel (true ou false)
  • aria-disabled: Inidica que um elemento é desabilitado e não pode ser interagido.
  • aria-describeby: Relaciona um elemento a um texto explicativo, geralmente usado para fornecer mais contexto para usuários de leitore de tela.
  • aria-pressed: Informa se um botão ou controle de alternância está pressionado ou não.
  • aria-live: Informa a prioridade de atualização dinâmica de uma região de conteúdo (exemplo: chat em tempo real).
  • aria-current: Indica o item atual em uma lista ou navegação (exemplo: link ativo em um menu).
  • aria-haspopup: Indica que um botão ou elemento pode abrir uma janela ou menu (exemplo: dropdown).
  • role: o papel de um elemento na interface, como "dialog", "button", "navigation".
  • aria-controls: Relaciona um elemento a outro que ele controla (como um botão que abre um modal).
  • aria-live: Define a prioridade de atualização de conteúdo dinâmico, com valores como "polite", "assertive".
  • aria-labelledby: Relaciona um elemento a outro que o descreve, geralmente utilizado em conjunto com títulos e descrições.
